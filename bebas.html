featured project

project BEBASPUSTAKA

The Library Clearance Information System is designed to digitalize and automate academic compensation (kompen) management for IT Department students.

• Project Overview

Bebas Pustaka website was developed to digitalize and automate the the event of kompensasi(kompen) tasks within the IT Department, which previously relied on inefficient manual procedures. By integrating APIs between the library database (OPAC) and the SIKOMPEN platform, this Laravel-based system eliminates the need for physical queues and slashes book-loan verification times to just one second. Consequently, it reduces the administrative workload for librarians during peak end-of-semester periods by over 90%.

• Core Features

The system features built-in automation logic to independently verify student book loan statuses without manual intervention:

  • Dashboard: Visualizes library book loan database trends, allowing users to customize filters according to their specific requirements.
  • Library Clearance Filtering: An automated background process triggered during the approval cycle to filter and isolate students who meet the pre-defined clearance criteria.
  • Library Clearance Approval: An approval mechanism designed to batch-process clearance status for all students who have fulfilled the requirements.

• Language & Core System Environment

The tech stack and architectural frameworks implemented throughout this platform include:

REACT MySQL NodeJS Tailwind

• Generated System Outputs

After passing machine verification, this platform issues the following ready-to-download clearance files:

  • Digital Receipt: A status list of library clearance and academic compensation (kompen) for students undergoing the clearance process.

Core Features Demo:

Dashboard:

-> Borrowed Books Data Visualization

A dashboard feature designed to visualize book loans filtered by study program and cohort year. Users can also customize the chart type, choosing between bar charts and pie charts.

-> Visitor Traffic Data Visualization

A dashboard feature built to visualize library visitor numbers, which can be filtered on a daily, monthly, or yearly basis according to the selected year.

Library Clearance & Compensation Filtering:

-> Classifying Students Lacking Library Clearance

A classification feature that identifies students who still have outstanding book loans, marks them as not yet cleared, and provides the specific reason for their uncleared status.

Library Clearance & Compensation Approval:

-> Individual Approval

An approval module that allows administrators to review and grant library clearance status to students one by one.

-> Bulk Approval

An approval module that allows administrators to grant library clearance status to all eligible students simultaneously using a bulk-approval feature.
